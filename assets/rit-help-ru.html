Быстрый старт

Для стабильного результата начинайте с компактного выделения таблицы: числовая область данных плюс, если нужно, ближайшие строки баннера сверху. Если выделение содержит текст вопроса, лишние комментарии или несколько таблиц сразу, результат может быть неполным или расчёт может остановиться с предупреждением.

ResearchSignal Table помогает работать с подготовленными research-таблицами, но не является полностью автоматическим распознавателем любых таблиц на листе. Для сложных макетов лучше сначала протестировать расчёт на одном фрагменте таблицы.

Основные кнопки в панели:

  • «Запустить»: надстройка прочитает значения из выделенного диапазона, найдёт лейблы строк слева, строки баннера сверху, определит типы метрик и рассчитает статистически значимые отличия.
  • «Очистить значимости»: удаляет добавленные маркеры и заливки из выделенного диапазона, по возможности восстанавливая очищенные числовые значения.

Поддерживаются доли/проценты, средние с SD или дисперсией, NPS по структуре Promoters/Detractors, а также NPS с SD или дисперсией. В сложной таблице может быть несколько блоков расчёта.

Данные таблицы обрабатываются локально в Excel WebView. Они не отправляются на сервер.

Основные настройки

Уровень значимости confidence level

Определяет строгость статистической проверки. Чем выше уровень, тем сложнее признать отличие значимым.

УровеньЧто означает
99%Очень строгий режим: значимыми будут только самые уверенные отличия.
95%Стандартный исследовательский режим по умолчанию.
90%, 80%, 66.6% Более мягкие режимы: подходят для разведочной проверки, но дают больше значимых отличий.

Уровень значимости применяется ко всем типам расчётов: процентам, средним, NPS, Тотал-сравнениям и сравнению с предыдущей колонкой.

Односторонний тест one-tailed

Переключает статистический порог с двустороннего теста на односторонний. При том же уровне значимости односторонний тест легче находит отличия.

Направление маркеров при этом не меняется: надстройка всё равно смотрит, какая колонка фактически выше или ниже.

95% двусторонний тест → z ≈ 1.96 95% односторонний тест → z ≈ 1.645
Используйте односторонний тест осознанно. Он уместен, когда заранее предполагается проверка различия в одном направлении, а не просто «хочется получить больше значимых отличий».
Округлять значения в ячейках

Округляет значения перед добавлением букв, стрелок или маркеров в ячейки. Расчёт значимости выполняется по исходным очищенным значениям, а не по округлённым отображаемым значениям.

Тип строкиПо умолчаниюЕсли настройка включена
Доли, NPS, Promoters, Detractors1 знакцелое значение
Mean, SD, Variance2 знака1 знак

Сравнение с предыдущей колонкой

Сравнение с предыдущей колонкой arrows

Меняет режим сравнения: каждая колонка сравнивается только с колонкой слева.

Колонка 2 vs Колонка 1 Колонка 3 vs Колонка 2 Колонка 4 vs Колонка 3

Вместо букв используются стрелки: ↑, если текущая колонка значимо выше предыдущей, и ↓, если значимо ниже. Стрелка ставится только в правую, текущую, колонку пары.

Этот режим автоматически отключает «Проставлять буквы в баннере» и несовместим с «Сравнивать только с Тотал».

Применить заливку

Отвечает за заливку ячеек со стрелками в режиме сравнения с предыдущей колонкой. При включении режима эта настройка включается автоматически, но её можно снять вручную.

  • Значимое повышение использует обычный цвет значимости.
  • Значимое снижение использует цвет для значений ниже Тотала.
Сравнение с предыдущей колонкой при учёте баннера

Если включён учёт структуры баннера, предыдущая колонка сравнивается только внутри той же группы баннера. Сравнение не пересекает границы групп.

Gender: Male vs Female Age: 18-24 vs 25-34 Не выполняется: 18-24 vs Female

Тотал-колонки в баннерном режиме не включаются в цепочку сравнения с предыдущей колонкой.

Баннер

Проставлять буквы в баннере

Добавляет буквенные индексы колонок в строку над выделенным диапазоном. Буквы баннера проставляются в видимые лейблы баннера, когда это возможно, и существуют отдельно от маркеров в ячейках с данными.

Segment 1 → Segment 1 (a) Segment 2 → Segment 2 (b)

Старые хвостовые маркеры в формате (a), (b) заменяются актуальными. Если колонка больше не должна иметь букву, старый маркер удаляется.

Если выделенный диапазон начинается с первой строки листа, буквы в баннер поставить некуда. В этом сценарии расчёт не выполнится.
Учитывать структуру баннера рекомендуется для кросс-табов

Включает баннерный движок. Надстройка анализирует строки над выделенным диапазоном, определяет группы колонок, локальные и глобальные Тоталы. Многоуровневые баннеры и баннеры с объединёнными ячейками поддерживаются лучше, но ResearchSignal Table не гарантирует корректную обработку любого сложного макета листа.

При включённой настройке:

  • колонки из разных групп баннера не сравниваются друг с другом;
  • буквы становятся локальными внутри группы;
  • Тотал определяется автоматически по структуре баннера;
  • ручные настройки расположения Тотала блокируются;
  • если дополнительно включено автоматическое определение волн, волновые группы могут перейти в режим сравнения с предыдущей колонкой.
Автоматически определять волны

Эта настройка появляется только при включённом «Учитывать структуру баннера» и по умолчанию выключена. Она разрешает надстройке распознавать волновые группы в баннере и автоматически применять к ним сравнение с предыдущей колонкой.

Если настройка выключена, группы с названиями вроде Wave, Period, Волна или Период считаются обычными группами баннера и сравниваются по стандартным внутригрупповым правилам.

Если настройка включена, автоматическое сравнение с предыдущей колонкой применяется только внутри распознанных волновых групп. Остальные группы продолжают использовать обычные сравнения внутри группы.

Лейблы значений слева листа

По умолчанию надстройка ищет названия строк непосредственно слева от выделенного диапазона. Эта настройка заставляет читать лейблы из самых левых колонок листа.

Это полезно для широких таблиц, где пользователь выделяет только правую часть таблицы, а названия строк остаются далеко слева.

Тотал

Сравнивать только с Тотал

Оставляет только сравнения с Тоталом. Обычные сравнения между сегментами не выполняются.

Если включён учёт структуры баннера, Тотал определяется автоматически. Если Тотал в баннере не найден, расчёт не создаст сравнений и покажет сообщение в статусе.

Настройка несовместима со сравнением с предыдущей колонкой.

Не сравнивать с Тотал

Полностью исключает Тотал из расчётов. Тотал не используется как референс и не сравнивается с другими колонками.

При учёте структуры баннера исключаются все найденные Тотал-колонки: и локальные, и глобальный.

Первая колонка — Тотал

Ручной режим для таблиц без анализа структуры баннера. Первая колонка выделенного диапазона считается Тоталом.

  • Тотал не получает обычную буквенную метку.
  • Колонки справа сравниваются с Тоталом.
  • Для значимо более высоких значений используется маркер T.
  • Для значимо более низких значений используется маркер t.

Если включён учёт структуры баннера, ручные настройки расположения Тотала блокируются, потому что Тотал определяется автоматически.

Тотал в каждом баннере

Настройка относится к сценариям с локальными Тоталами внутри групп. В текущей логике при включённом баннерном движке такие Тоталы определяются автоматически по структуре баннера.

Локальный и глобальный Тотал

Если глобального Тотала нет, локальный Тотал используется как референс внутри своей группы.

Если глобальный Тотал найден, он становится единственным Тотал-референсом. Локальные Тоталы не используются как референсы для сегментов, но сами могут сравниваться с глобальным Тоталом и получать маркеры T или t в ячейках.

Тотал-колонки никогда не получают обычные буквенные маркеры в баннере.

Заливки

Обычная значимость

Цвет для ячеек со статистически значимыми положительными отличиями. По умолчанию используется светло-зелёная заливка.

Заливка < Тотал

Цвет для ячеек, которые значимо ниже Тотала, а также для значимого снижения в режиме сравнения с предыдущей колонкой.

Заливка только для Тотала

Ограничивает обычную зелёную заливку: она применяется только к ячейкам, которые значимо выше Тотала. Отличия между обычными сегментами могут по-прежнему получать буквы, но без зелёной заливки.

Настройка отключается в режиме сравнения с предыдущей колонкой.

Приоритет заливок

Если на одну ячейку претендует несколько типов заливки, применяется приоритет:

  1. маленькая база;
  2. значимо ниже Тотала / ниже предыдущей колонки;
  3. обычная значимость;
  4. без заливки.

Маленькие базы

Не сравнивать маленькие базы

Если база колонки меньше заданного порога, эта колонка исключается из расчёта до выполнения статистических тестов. Это помогает избежать нестабильных сравнений на небольших выборках.

Для такой колонки не появляются маркеры значимости. Заливка маленькой базы применяется ко всей колонке внутри текущего расчётного блока, включая саму строку базы.

База <

Порог маленькой базы. Например, если указано 50, то колонки с базой 49 и ниже будут исключены.

Заливка маленькой базы

Цвет, которым помечаются колонки с маленькой базой. Эта заливка имеет самый высокий приоритет.

Если в ручном режиме «Первая колонка — Тотал» у Тотала обнаружена маленькая база, расчёт останавливается. Это защитное правило: Тотал не должен использоваться как референс, если его база ниже заданного порога.

Маркеры значимости

МаркерЧто означает
a, b, c... Обычные буквенные маркеры. Показывают, относительно какой колонки значение значимо выше.
TЗначение значимо выше Тотала.
tЗначение значимо ниже Тотала.
↑Значение значимо выше предыдущей колонки.
↓Значение значимо ниже предыдущей колонки.

Обычные буквы t, T, кириллические т, Т исключены из алфавита буквенных маркеров, чтобы не конфликтовать с Тотал-маркерами.

Числовой вывод

Ячейки с добавленным текстом маркера, например 21% b, могут стать текстом, так как Excel не может хранить их как чистые числа. Ячейки без маркеров по возможности остаются числовыми.

ResearchSignal Table старается сохранять форматы отображения:

  • 28 остаётся обычным числом;
  • 28% остаётся процентом;
  • 0.28 остаётся десятичной долей.

Очистка значимости по возможности возвращает ячейкам исходный числовой вид.

NPS и другие метрики

Строки NPS получают специальные маркеры значимости для NPS. Строки поддержки Promoters, Detractors и Neutral могут получать обычные маркеры для долей, если это применимо.

Строки с базой и строки SD/дисперсии не получают маркеров значимости.

Статусы и предупреждения

После запуска надстройка показывает короткий пользовательский статус. Техническая диагностика баннера скрыта.

Обычный успешный запуск выглядит так:

Расчёт выполнен. Обработано блоков: 1.

Дополнительные сообщения появляются только в важных случаях, например:

  • найден глобальный Тотал;
  • для волновой группы автоматически применено сравнение с предыдущей колонкой;
  • включено сравнение только с Тоталом, но Тотал не найден;
  • в одной группе найдено несколько Тоталов;
  • над выделенным диапазоном нет строки для анализа баннера;
  • выделение расположено в первой строке, а включена запись букв в баннер.
