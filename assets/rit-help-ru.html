Быстрый старт

Для работы надстройки рекомендуется выделять только числовую область данных. Не включайте в выделение названия строк слева, строки баннера/заголовков сверху или текст вопроса.

При подозрительном выделении надстройка может показать предупреждение. Полномасштабное автоматическое распознавание всей таблицы по одному клику сейчас не поддерживается.

Кнопки находятся в верхней части панели:

  • «Запустить расчёт»: надстройка прочитает значения из выделенного диапазона, найдёт лейблы строк слева, строки баннера сверху, определит типы метрик и рассчитает статистически значимые отличия.
  • «Очистить значимость»: удаляет маркеры и заливки из выделенного диапазона, по возможности восстанавливая очищенные числовые значения.

Поддерживаются доли/проценты, средние с SD или дисперсией, NPS по структуре Promoters/Detractors, а также NPS с SD или дисперсией. В сложной таблице может быть несколько блоков расчёта.

Данные таблицы обрабатываются локально в Excel WebView. Они не отправляются на сервер.

Основные настройки

Уровень значимости confidence level

Определяет строгость статистической проверки. Чем выше уровень, тем сложнее признать отличие значимым.

Уровень Что означает
99% Очень строгий режим: значимыми будут только самые уверенные отличия.
95% Стандартный исследовательский режим по умолчанию.
90%, 80%, 66.6% Более мягкие режимы: подходят для быстрой разведочной проверки, но дают больше значимых отличий.

Уровень значимости применяется ко всем типам расчётов: процентам, средним, NPS, Total-сравнениям и сравнению с предыдущей колонкой.

Односторонний тест one-tailed

Переключает статистический порог с двустороннего теста на односторонний. При том же уровне значимости односторонний тест легче находит отличия.

Направление маркеров при этом не меняется: надстройка всё равно смотрит, какая колонка фактически выше или ниже.

95% двусторонний тест → z ≈ 1.96 95% односторонний тест → z ≈ 1.645
Используйте односторонний тест осознанно. Он уместен, когда заранее предполагается проверка различия в одном направлении, а не просто «хочется получить больше значимых отличий».
Округлять значения в ячейках

Округляет значения перед добавлением букв, стрелок или маркеров в ячейки. Расчёт значимости выполняется по исходным очищенным значениям, а не по округлённым отображаемым значениям.

Тип строки По умолчанию Если настройка включена
Доли, NPS, Promoters, Detractors 1 знак целое значение
Mean, SD, Variance 2 знака 1 знак

Сравнение с предыдущей колонкой

Сравнение с предыдущей колонкой arrows

Меняет режим сравнения: каждая колонка сравнивается только с колонкой слева.

Колонка 2 vs Колонка 1 Колонка 3 vs Колонка 2 Колонка 4 vs Колонка 3

Вместо букв используются стрелки: ↑, если текущая колонка значимо выше предыдущей, и ↓, если значимо ниже. Стрелка ставится только в правую, то есть текущую, колонку пары.

Этот режим автоматически отключает «Проставлять буквы в баннере» и несовместим с «Сравнивать только с Total».

Применить заливку

Отвечает за заливку ячеек со стрелками в режиме сравнения с предыдущей колонкой. При включении режима сравнения с предыдущей колонкой эта настройка включается автоматически, но её можно снять вручную.

  • Значимое повышение использует обычный цвет значимости.
  • Значимое снижение использует цвет для значений ниже Total.
Сравнение с предыдущей колонкой при учёте баннера

Если включён учёт структуры баннера, предыдущая колонка сравнивается только внутри той же группы баннера. Сравнение не пересекает границы групп.

Gender: Male vs Female Age: 18-24 vs 25-34 Не выполняется: 18-24 vs Female

Total-колонки в баннерном режиме не включаются в цепочку сравнения с предыдущей колонкой.

Баннер

Проставлять буквы в баннере

Добавляет буквенные индексы колонок в строку над выделенным диапазоном. Буквы баннера проставляются в видимые лейблы баннера, когда это возможно, и существуют отдельно от маркеров в ячейках с данными.

Segment 1 → Segment 1 (a) Segment 2 → Segment 2 (b)

Старые хвостовые маркеры в формате (a), (b) заменяются актуальными. Если колонка больше не должна иметь букву, старый маркер удаляется.

Если выделенный диапазон начинается с первой строки листа, буквы в баннер поставить некуда. В этом сценарии расчёт не выполнится.
Учитывать структуру баннера рекомендуется для кросс-табов

Включает баннерный движок. Надстройка анализирует строки над выделенным диапазоном, определяет группы колонок, локальные и глобальные Total. Многоуровневые баннеры и баннеры с объединёнными ячейками теперь поддерживаются лучше. Однако надстройка не гарантирует поддержку абсолютно любого сложного макета или автоматическое распознавание всей структуры листа. Автоматическое распознавание волновых структур включается отдельной настройкой.

При включённой настройке:

  • колонки из разных групп баннера не сравниваются друг с другом;
  • буквы становятся локальными внутри группы;
  • Total определяется автоматически по структуре баннера;
  • ручные настройки расположения Total блокируются;
  • если дополнительно включено автоматическое определение волн, волновые группы могут перейти в режим сравнения с предыдущей колонкой.
Автоматически определять волны

Эта настройка появляется только при включённом «Учитывать структуру баннера» и по умолчанию выключена. Она разрешает надстройке распознавать волновые группы в баннере и автоматически применять к ним сравнение с предыдущей колонкой.

Если настройка выключена, группы с названиями вроде Wave, Period, Волна или Период считаются обычными группами баннера и сравниваются по стандартным внутригрупповым правилам.

Если настройка включена, автоматическое сравнение с предыдущей колонкой применяется только внутри распознанных волновых групп. Остальные группы продолжают использовать обычные сравнения внутри группы. Состояние основного чекбокса «Сравнение с предыдущей колонкой» при этом визуально не меняется.

Лейблы значений слева листа

По умолчанию надстройка ищет названия строк непосредственно слева от выделенного диапазона. Эта настройка заставляет читать лейблы из самых левых колонок листа.

Это полезно для широких таблиц, где пользователь выделяет только правую часть таблицы, а названия строк остаются далеко слева.

Волновые баннеры

Если включена настройка «Автоматически определять волны» и надстройка распознаёт группу как волновую, внутри этой группы автоматически применяется сравнение с предыдущей колонкой. Остальные группы продолжают использовать обычные внутригрупповые сравнения.

Сейчас волновая группа определяется по верхнему лейблу группы:

wave, waves, волна, волны, period, periods, период, периоды, замер, замеры

Простые числовые подписи вроде 1, 2, 3 не считаются признаком волновой структуры, потому что это могут быть номера сегментов или объектов.

Для автоматически распознанных волновых групп буквы в баннер не подставляются, даже если настройка «Проставлять буквы в баннере» включена. В ячейках используются стрелки.

Total

Сравнивать только с Total

Оставляет только сравнения с Total. Обычные сравнения между сегментами не выполняются.

Если включён учёт структуры баннера, Total определяется автоматически. Если Total в баннере не найден, расчёт не создаст сравнений и покажет сообщение в статусе.

Настройка несовместима со сравнением с предыдущей колонкой.

Не сравнивать с Total

Полностью исключает Total из расчётов. Total не используется как референс и не сравнивается с другими колонками.

При учёте структуры баннера исключаются все найденные Total-колонки: и локальные, и глобальный.

Первая колонка — Total

Ручной режим для таблиц без анализа структуры баннера. Первая колонка выделенного диапазона считается Total.

  • Total не получает обычную буквенную метку.
  • Колонки справа сравниваются с Total.
  • Для значимо более высоких значений используется маркер T.
  • Для значимо более низких значений используется маркер t.

Если включён учёт структуры баннера, ручные настройки расположения Total блокируются, потому что Total определяется автоматически.

Total в каждом баннере

Настройка относится к сценариям с локальными Total внутри групп. В текущей логике при включённом баннерном движке такие Total определяются автоматически по структуре баннера.

Локальный и глобальный Total

Если глобального Total нет, локальный Total используется как референс внутри своей группы.

Если глобальный Total найден, он становится единственным Total-референсом. Локальные Total не используются как референсы для сегментов, но сами могут сравниваться с глобальным Total и получать маркеры T или t в ячейках.

Total-колонки никогда не получают обычные буквенные маркеры в баннере.

Заливки

Цвет значимости

Цвет для ячеек со статистически значимыми положительными отличиями. По умолчанию используется светло-зелёная заливка.

Цвет ниже Total

Цвет для ячеек, которые значимо ниже Total, а также для значимого снижения в режиме сравнения с предыдущей колонкой.

Заливка только для Total

Ограничивает обычную зелёную заливку: она применяется только к ячейкам, которые значимо выше Total. Отличия между обычными сегментами могут по-прежнему получать буквы, но без зелёной заливки.

Настройка отключается в режиме сравнения с предыдущей колонкой.

Приоритет заливок

Если на одну ячейку претендует несколько типов заливки, применяется приоритет:

  1. маленькая база;
  2. значимо ниже Total / ниже предыдущей колонки;
  3. обычная значимость;
  4. без заливки.

Маленькие базы

Исключать маленькие базы

Если база колонки меньше заданного порога, эта колонка исключается из расчёта до выполнения статистических тестов. Это помогает избежать нестабильных сравнений на небольших выборках.

Для такой колонки не появляются маркеры значимости. Заливка маленькой базы применяется ко всей колонке внутри текущего расчётного блока, включая саму строку базы.

База <

Порог маленькой базы. Например, если указано 50, то колонки с базой 49 и ниже будут исключены.

Цвет маленькой базы

Цвет, которым помечаются колонки с маленькой базой. Эта заливка имеет самый высокий приоритет.

Если в ручном режиме «Первая колонка — Total» у Total обнаружена маленькая база, расчёт останавливается. Это защитное правило: Total не должен использоваться как референс, если его база ниже заданного порога.

Сохранение настроек

Сохранять настройки

Управляет сохранением текущих настроек панели.

Режим Поведение
Не сохранять Настройки не сохраняются между запусками.
Локально Настройки сохраняются в localStorage текущего Excel WebView / браузера.

Возвращает настройки к значениям по умолчанию и удаляет локально сохранённые настройки.

Маркеры значимости

Маркер Что означает
a, b, c... Обычные буквенные маркеры. Показывают, относительно какой колонки значение значимо выше.
T Значение значимо выше Total.
t Значение значимо ниже Total.
↑ Значение значимо выше предыдущей колонки.
↓ Значение значимо ниже предыдущей колонки.

Обычные буквы t, T, кириллические т, Т исключены из алфавита буквенных маркеров, чтобы не конфликтовать с Total-маркерами.

Числовой вывод

Ячейки с добавленным текстом маркера (например, 21% b) могут стать текстом, так как Excel не может хранить их как чистые числа. Ячейки без маркеров по возможности остаются числовыми.

Надстройка сохраняет форматы отображения:

  • 28 остаётся обычным числом;
  • 28% остаётся процентом;
  • 0.28 остаётся десятичной долей.

Очистка значимости по возможности возвращает ячейкам исходный числовой вид.

NPS и другие метрики

Строки NPS получают специальные маркеры значимости для NPS. Строки поддержки (Promoters, Detractors, Neutral) могут получать обычные маркеры для долей, если это применимо.

Строки с базой и строки SD/дисперсии не получают маркеров значимости.

Статусы и предупреждения

После запуска надстройка показывает короткий пользовательский статус. Техническая диагностика баннера скрыта.

Обычный успешный запуск выглядит так:

Расчёт выполнен. Обработано блоков: 1.

Дополнительные сообщения появляются только в важных случаях, например:

  • найден глобальный Total;
  • для волновой группы автоматически применено сравнение с предыдущей колонкой, если включено автоматическое определение волн;
  • включено сравнение только с Total, но Total не найден;
  • в одной группе найдено несколько Total;
  • над выделенным диапазоном нет строки для анализа баннера;
  • выделение расположено в первой строке, а включена запись букв в баннер.
