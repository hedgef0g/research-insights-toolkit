# User Guide

Практическая справка по текущему MVP Research Insights Toolkit (RIT) / ResearchSignal Table.

Этот документ описывает только текущее поведение продукта. Если поведение в Excel и в этом файле расходится, ориентируйтесь на фактический интерфейс текущей сборки и сообщите о расхождении.

## Что делает RIT

RIT помогает проверять статистическую значимость прямо в Excel-таблицах без ручной подготовки данных во внешних пакетах.

Текущий MVP умеет:

- рассчитывать значимости в таблице;
- очищать ранее проставленные значимости;
- проверять таблицу или книгу в режиме без изменения данных;
- формировать служебные листы `Content` и `Run report`.

## Какую таблицу ожидает RIT

RIT лучше всего работает с исследовательскими таблицами, где:

- числовые данные находятся в одной непрерывной прямоугольной области;
- подписи строк находятся слева от числовой области;
- баннер или заголовки колонок находятся сверху;
- внутри одной таблицы нет пустой строки, разделяющей её на несколько отдельных таблиц;
- активная ячейка для режимов `Текущая таблица` стоит внутри распознаваемой таблицы.

Для ручного режима `Расчёт` safest path такой:

1. Выделите одну непрерывную числовую область таблицы.
2. Не включайте в выделение две отдельные таблицы.
3. Не используйте Ctrl+Click для несмежных диапазонов.

## Главное различие: выделение против активной ячейки

Это ключевая разница в текущем MVP:

- `Расчёт` работает по выделенному диапазону.
- `Проверка -> Текущая таблица` и `Автозапуск -> Текущая таблица` работают по таблице вокруг активной ячейки.

Это значит:

- если курсор стоит в таблице A, а выделен диапазон таблицы B, то `Расчёт` берёт таблицу B;
- `Проверка -> Текущая таблица` и `Автозапуск -> Текущая таблица` в той же ситуации берут таблицу A;
- широкое выделение само по себе не превращает `Текущая таблица` в режим обработки нескольких таблиц.

## `Расчёт`

`Расчёт` предназначен для ручной работы с конкретным выделенным диапазоном.

### `Рассчитать`

Используйте `Рассчитать`, когда вы уже выделили нужную таблицу или числовую область и хотите записать значимости обратно в Excel.

Практически это означает:

- RIT использует текущее выделение, а не таблицу вокруг активной ячейки;
- выделение должно быть одним непрерывным диапазоном;
- слишком широкие выделения, захватывающие несколько таблиц, блокируются;
- неподдерживаемые несмежные выделения показывают контролируемое сообщение и не запускают частичную обработку.

### `Очистить значимости`

Используйте `Очистить значимости`, чтобы убрать маркеры и связанную с ними служебную разметку из текущего выбранного диапазона.

Важно:

- этот режим тоже зависит от выделенного диапазона;
- для него действуют те же ограничения, что и для `Рассчитать`;
- если выделение небезопасно или неоднозначно, очистка не должна идти "наугад".

### `Проверить выделение`

`Проверить выделение` делает проверку без записи результатов в таблицу.

Используйте его, когда хотите быстро понять:

- похоже ли текущее выделение на поддерживаемую таблицу;
- нашлись ли блоки данных и базы;
- есть ли предупреждения или критические проблемы до расчёта.

### Что важно помнить в `Расчёт`

- Поддерживается только одно непрерывное выделение.
- Broad multi-table ranges с пустыми строками между таблицами блокируются.
- Ctrl+Click / non-contiguous selection не поддерживается.
- `Расчёт` и `Текущая таблица` в других разделах намеренно работают по разным правилам.

### Диагностический лист в пакетных режимах

Для пакетных режимов `Автозапуск -> Текущий лист` и `Автозапуск -> Вся книга` может показываться флажок `Добавить диагностический лист`.

Если он включён, RIT создаёт или обновляет служебный лист `Run report` с итогами обработки таблиц. Это удобно, когда нужно не только записать значимости, но и получить список `processed`, `skipped`, `blocked` и `error` по найденным таблицам.

Для `Расчёт` по выделенному диапазону этот флажок не используется: в этом режиме расчёт идёт только по текущему выделению.

## `Проверка`

`Проверка` по умолчанию работает в режиме read-only: она не меняет исходные таблицы.

Исключение только одно:

- если включён флажок `Записать результат`, RIT может создать или обновить служебный лист `Run report`.

### `Текущая таблица`

`Проверка -> Текущая таблица` использует таблицу вокруг активной ячейки.

Используйте этот режим, когда:

- вы хотите проверить именно ту таблицу, в которой стоит курсор;
- выделение не подготовлено или не должно влиять на результат;
- нужно быстро увидеть диагностику по одной таблице без записи значимостей.

Типичные сообщения:

- таблица не найдена;
- активная ячейка стоит на служебном листе `Content` или `Run report`;
- найдена неоднозначная или неподдерживаемая табличная область;
- использовано несмежное выделение, которое продукт не поддерживает.

### `Текущий лист`

`Проверка -> Текущий лист` проверяет распознанные таблицы на активном листе.

Это удобно, когда:

- на листе несколько таблиц;
- нужно понять, какие таблицы будут обработаны, а какие пропущены;
- вы хотите получить обзор без записи значимостей в сами таблицы.

### `Вся книга`

`Проверка -> Вся книга` проверяет распознанные таблицы по всей книге.

Используйте этот режим, когда нужно:

- получить обзор по нескольким листам;
- понять, какие таблицы поддерживаются;
- собрать диагностический `Run report` по книге без изменения исходных таблиц.

## `Автозапуск`

`Автозапуск` использует распознанные таблицы и может сразу выполнять расчёт или очистку.

### `Текущая таблица -> Рассчитать`

Этот режим работает по активной ячейке и найденной вокруг неё таблице.

Он полезен, когда:

- вы не хотите руками выделять числовую область;
- активная ячейка уже стоит внутри нужной таблицы;
- вы хотите быстро запустить расчёт по одной распознанной таблице.

### `Текущая таблица -> Очистить значимости`

Работает так же, но очищает значимости в таблице, найденной вокруг активной ячейки.

### `Текущий лист -> Рассчитать / Очистить значимости`

Эти команды проходят по поддерживаемым распознанным таблицам на активном листе.

Они подходят, когда:

- на листе несколько таблиц;
- вы хотите пакетно посчитать или очистить только активный лист;
- вы готовы к тому, что сомнительные кандидаты могут быть пропущены или заблокированы.

### `Вся книга -> Рассчитать / Очистить значимости`

Эти команды проходят по поддерживаемым распознанным таблицам во всей книге.

Это пакетный режим для книги, а не для произвольного выделения.

Главное ограничение `Автозапуск`:

- `Текущая таблица` зависит от активной ячейки;
- `Текущий лист` и `Вся книга` зависят от распознавания таблиц;
- служебные листы `Content` и `Run report` не считаются исходными таблицами и не должны обрабатываться как источник данных.

## `Настройки`

Панель `Настройки` находится под основными рабочими блоками и содержит пять вкладок: `Основные`, `Сравнения`, `Баннер`, `Базы`, `Дизайн`.

Панель можно свернуть и развернуть кнопкой `Настройки`. Состояние панели запоминается локально на этом устройстве, поэтому после следующего открытия add-in она обычно возвращается в том же виде.

### `Основные`

Во вкладке `Основные` задаются базовые параметры расчёта:

- `Уровень значимости` выбирает порог проверки: `99%`, `95%`, `90%`, `80%` или `66.6%`;
- `Односторонний тест` переключает тест в one-tailed режим;
- `База для расчёта` задаёт предпочтительный тип базы, если в таблице есть несколько вариантов строк базы.

В селекторе базы сейчас видны варианты `Авто`, `Effective Base`, `Unweighted Base`, `Base`, `Weighted Base`. На практике безопаснее оставлять `Авто`, если у вас нет явной причины принудительно выбирать конкретный тип базы из уже видимых в таблице строк.

### `Сравнения`

Во вкладке `Сравнения` управляются правила сравнения колонок:

- `Сравнение с предыдущей колонкой` добавляет проверку соседних колонок слева направо;
- `Применить заливку` становится доступным для предыдущей опции и добавляет фоновую подсветку для таких сравнений;
- `Сравнивать только с Тоталом` оставляет сравнения только относительно Total;
- `Не сравнивать с Тоталом` исключает Total из сравнений;
- `Первая колонка — Тотал` подсказывает продукту, что первый столбец нужно трактовать как Total;
- `Тотал в каждом баннере` подсказывает RIT искать и учитывать Total внутри каждого баннерного блока, а не только один общий Total.

`Сравнивать только с Тоталом` и `Не сравнивать с Тоталом` взаимоисключаемы: обычно выбирают что-то одно в зависимости от логики таблицы.

### `Баннер`

Во вкладке `Баннер` настраивается работа с заголовками колонок:

- `Проставлять буквы в баннере` добавляет буквенные метки колонок над таблицей;
- `Учитывать структуру баннера` помогает уважать группы и иерархию в сложных баннерах;
- `Автоматически определять волны` включает автоопределение wave/banner-структур, если они читаются по заголовкам;
- `Лейблы значений слева листа` сообщает, что подписи строк находятся слева от числовой области листа.

Практически это значит, что для простых таблиц часто достаточно минимальных настроек, а для многострочных или групповых баннеров имеет смысл включать `Учитывать структуру баннера` и связанные с ним опции.

### `Базы`

Во вкладке `Базы` находятся ограничения по малым базам:

- `Не сравнивать маленькие базы` исключает из сравнений колонки с недостаточной базой;
- `База <` задаёт порог малой базы числом.

Если эта логика вам нужна, проверьте, что порог соответствует стандарту проекта. Если не нужна, флажок можно оставить выключенным.

### `Дизайн`

Во вкладке `Дизайн` собраны настройки вывода:

- `Округлять значения в ячейках` округляет отображаемые значения после записи;
- `Обычная значимость` задаёт цвет заливки для обычных значимых отличий;
- `Заливка < Тотал` задаёт отдельный цвет для отличий ниже Total;
- `Заливка только для Тотала` ограничивает заливку только total-сравнениями;
- `Заливка маленькой базы` задаёт цвет подсветки для малых баз.

Эти настройки меняют оформление результата, но не предназначены для изменения статистической логики.

### Сохранение настроек и нижняя панель

Внизу taskpane есть служебная строка с тремя элементами:

- режим `Сохранять настройки...`;
- кнопка `Сброс`;
- ссылка `Справка об использовании`.

Режимы сохранения сейчас такие:

- `Не сохранять` — настройки не хранятся между сессиями;
- `Локально` — настройки сохраняются в локальном хранилище на этом устройстве;
- `В облаке` — опция видна, но в текущем UI ещё недоступна.

Кнопка `Сброс` возвращает настройки к значениям по умолчанию для текущей версии интерфейса.

## `Оглавление`

`Оглавление` работает только в контексте книги.

Что делает этот режим:

- создаёт лист `Content`, если его ещё нет;
- обновляет `Content`, если он уже существует;
- перечисляет найденные исходные таблицы;
- ставит ссылки на таблицы в колонке с заголовком таблицы;
- использует безопасный fallback `Таблица N`, если надёжный заголовок не найден;
- может поставить обратные ссылки, если включена соответствующая опция.

### Формат `Оглавления`

Во вкладке `Оглавление` есть селектор `Формат оглавления` и флажок `Ставить обратные ссылки`.

Селектор формата помогает выбрать, сколько служебной информации нужно в листе `Content`:

- `Для клиента` — самый простой и аккуратный вариант, когда нужен в первую очередь читабельный список таблиц;
- `С минимальной проверкой` — компромиссный режим, когда нужен список таблиц и базовая диагностическая информация;
- `С полной проверкой` — самый подробный режим, когда `Content` используется ещё и как рабочий обзор по качеству таблиц.

`Ставить обратные ссылки` добавляет навигационные ссылки из `Content` обратно к найденным таблицам книги.

Что не нужно ожидать:

- `Content` не является исходной таблицей для расчётов;
- баннерное слово вроде `Всего` не должно использоваться как заголовок таблицы, если нормального заголовка нет.

## `Run report`

`Run report` — это служебный диагностический лист.

Когда он появляется:

- при `Автозапуск -> Текущий лист` или `Автозапуск -> Вся книга`, если включён `Добавить диагностический лист`;
- при `Проверка`, если включён `Записать результат`;
- в пакетных режимах, где текущая сборка записывает диагностический отчёт;
- при повторном запуске обычно обновляется существующий лист, а не создаётся новая копия.

Что можно увидеть в `Run report`:

- `processed` или эквивалентный успешный результат: таблица была обработана;
- `checked`: таблица была проверена без изменения исходных данных;
- `skipped`: кандидат был пропущен;
- `blocked`: обработка остановлена контролируемым правилом;
- `error`: произошла ошибка.

Как заполняется название:

- если у таблицы найден понятный заголовок, используется он;
- если надёжного заголовка нет, используется безопасный fallback вида `Таблица N`.

Что важно помнить:

- `Run report` сам не является исходной таблицей;
- `Content` и `Run report` должны игнорироваться или блокироваться в режимах обработки таблиц;
- строки отчёта помогают понять, что было обработано, проверено, пропущено, заблокировано или завершилось ошибкой.

## Когда что использовать

- Используйте `Расчёт`, когда хотите точно управлять целевым диапазоном через выделение.
- Используйте `Проверка`, когда хотите read-only диагностику по текущей таблице, листу или книге.
- Используйте `Автозапуск`, когда хотите работать по распознанным таблицам без ручного выделения.
- Используйте `Оглавление`, когда нужен список таблиц книги и ссылки на них в `Content`.

## Known limitations

- Для ручных действий через `Расчёт` поддерживается только одно непрерывное выделение.
- `Расчёт` работает по выделенному диапазону, а `Текущая таблица` в `Проверка` и `Автозапуск` работает по таблице вокруг активной ячейки.
- Ctrl+Click / несмежные выделения не поддерживаются.
- Если выделение охватывает несколько отдельных таблиц, операция может быть заблокирована; для таких случаев лучше использовать режимы по `Текущий лист` или `Вся книга`.
- `Проверка -> Текущая таблица` и `Автозапуск -> Текущая таблица` зависят от того, стоит ли активная ячейка внутри распознаваемой таблицы.
- Сильно нерегулярные таблицы могут требовать ручной режим `Расчёт` вместо detected-table flows.
- Ожидаемые значимости зависят от корректного распознавания баз, блоков метрик и структуры таблицы.
- Автоматические режимы по листу и книге обрабатывают только поддерживаемые распознанные таблицы и могут пропускать неуверенные кандидаты.
- Служебные листы `Content` и `Run report` намеренно не считаются исходными таблицами и не должны использоваться как источник данных для расчёта.
- Google Sheets UI и панель Google Sheets не входят в текущий MVP.
- Локализация продукта пока не завершена полностью.

## Troubleshooting

### Я выделил два диапазона через Ctrl+Click и получил сообщение о неподдерживаемом выделении

Текущий MVP поддерживает только одно непрерывное выделение для ручных действий. Снимите множественное выделение и выберите один прямоугольный диапазон.

### Я выделил сразу несколько таблиц и расчёт не запустился

Это ожидаемо для широких multi-table выделений. `Расчёт` рассчитан на один непрерывный рабочий диапазон, а не на произвольный набор таблиц в одном выделении. Если нужно обработать несколько таблиц, используйте `Текущий лист` или `Вся книга`.

### `Текущая таблица` проверяет не ту таблицу

Для режимов `Проверка -> Текущая таблица` и `Автозапуск -> Текущая таблица` важна активная ячейка, а не выделение. Поставьте курсор внутрь нужной таблицы и повторите действие.

### Появляется сообщение `No table found` или таблица не находится

Обычно это значит, что активная ячейка стоит вне распознаваемой таблицы или структура таблицы слишком неоднозначна. Попробуйте поставить курсор внутрь таблицы или использовать `Расчёт` с ручным выделением числовой области.

### `Content` или `Run report` игнорируется

Это ожидаемое поведение. Оба листа являются служебными и не должны обрабатываться как исходные исследовательские таблицы.

### Буквы в баннере выглядят устаревшими после повторного запуска

Сначала повторите расчёт для той же таблицы или используйте `Очистить значимости`, если нужно убрать прежнюю разметку. Если следы прошлого запуска сохраняются, зафиксируйте пример для smoke-проверки и заведите баг.

### В `Run report` заголовок выглядит как `Таблица N`

Это безопасный fallback. Он появляется, когда продукт не нашёл надёжный пользовательский заголовок таблицы и не должен использовать баннерную подпись вроде `Всего` вместо заголовка.

### Значимости стоят не там, где я ожидал

Проверьте:

- выделена ли именно нужная числовая область;
- не перепутаны ли режимы выделения и активной ячейки;
- корректно ли определились базы и блоки метрик;
- не была ли таблица частично пропущена или заблокирована.

Если проблема повторяется, сохраните пример и сверяйтесь со структурными ограничениями продукта.

## Язык интерфейса / Interface language

В правом верхнем углу брендинговой секции панели задач находится переключатель языка: **Русский** (по умолчанию) и **English**.

При переключении все видимые подписи кнопок, вкладок, подсказок и секций настроек обновляются без перезагрузки.

Выбор языка сохраняется в `localStorage` (`rit.ui.lang`) независимо от режима сохранения настроек.

**Текущее покрытие локализации (версия #229):**

- Вкладки действий: Расчёт / Run, Проверка / Check, Автозапуск / Autorun, Оглавление / Contents
- Кнопки области действия: Текущая таблица / Current table, Текущий лист / Current sheet, Вся книга / Whole workbook
- Основные кнопки действий: Рассчитать, Очистить значимости, Проверить выделение / таблицу / лист / книгу, Создать оглавление
- Подсказки рабочего пространства и суффиксы режима проверки
- Вкладки и ключевые метки секции настроек
- Метки панелей статуса и проверки
- Системные сообщения: несмежное выделение

**Намеренно не локализовано в этой версии:**

- Сгенерированные имена листов: `Content` и `Run report` — **стабильны и не изменяются**
- Заголовки колонок отчётов и диагностика в листе `Run report`
- Колонки листа `Content`
- Подробные диагностические сообщения о структуре и качестве таблицы
- Тексты подсказок к настройкам (тултипы)
- Некоторые статусные строки в глубоких сценариях автозапуска
